# 설계도 완전성 검증 체크리스트 (매 작업 시 자동 적용)

> 이 steering은 `inclusion: auto`로 설정되어 모든 대화에 자동 로드됩니다.
> KIRO가 `services/*/docs/` 하위에 산출물을 생성·수정할 때마다 아래 검증을 수행하세요.
> 작업자가 매번 요청하지 않아도 KIRO가 자동으로 체크합니다.

---

## A. 산출물 생성 시 필수 검증 (매 Step 완료 시)

### A-1. 코드 기반 완전성 검증
- [ ] 2-Pass 스캔을 수행했는가? (Pass 1: 구조 스캔 → Pass 2: 정밀 분석)
- [ ] ❌ 판정 전 6회 검색 최소 검증을 수행했는가? (grepSearch 3회 + fileSearch 2회 + 설정파일 1회)
- [ ] 누락 검증 로그 테이블을 산출물 하단에 포함했는가?
- [ ] 멀티 모듈(3개+) 서비스에서 모듈별 분할 분석을 수행했는가?

### A-2. 사용자 추가 정보 반영 검증
- [ ] `runtime-data.md` (수작업 수집 런타임 데이터)가 존재하면 해당 내용을 산출물에 반영했는가?
- [ ] 작업자가 채팅에서 추가로 알려준 정보(인터뷰 결과, 운영 환경 등)를 산출물에 반영했는가?
- [ ] 반영한 추가 정보에 `[작업자 제공]` 또는 `[인터뷰 결과]` 태그를 붙였는가?
- [ ] `glossary.md` 용어 사전의 정규화된 명칭을 사용했는가?

### A-3. Step 간 교차 검증
- [ ] 이전 Step 산출물과 ID/명칭/경로 참조가 일치하는가?
- [ ] 교차 검증 로그 테이블을 산출물 하단에 포함했는가?
- [ ] 불일치 발견 시 이전 산출물을 보정하고 `[교차검증 보정]` 태그를 붙였는가?

### A-4. 설계도 양식 완전성 검증
- [ ] 모든 항목에 확인 상태(✅/⚠️/❌/🔍)를 표기했는가?
- [ ] 모든 항목에 분석 주체 태그([KIRO]/[KIRO+도구]/[KIRO+확정]/[인터뷰])를 표기했는가?
