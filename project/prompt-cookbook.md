# 설계도 파묘 — 프롬프트 쿡북 (Prompt Cookbook)

> 작업자들이 KIRO와의 추가 프롬프팅으로 발견한 효과적인 패턴을 축적하는 문서입니다.
> 검증된 패턴은 steering 문서의 "💡 효과적인 추가 프롬프트 예시" 섹션에 반영됩니다.

---

## 사용법

1. 작업 완료 시 KIRO가 피드백을 요청하면, 효과적이었던 프롬프트를 공유하세요
2. KIRO가 아래 형식으로 자동 기록합니다
3. PM이 주기적으로 리뷰하여 ⭐(검증됨) 마크를 부여하고 steering에 반영합니다

---

## 기록 형식

```markdown
### [날짜] Step XX — {제목}
- **서비스**: {서비스명}
- **상황**: {어떤 항목이 누락/오판되었는지}
- **효과적 프롬프트**: {실제 사용한 프롬프트}
- **발견 결과**: {프롬프트로 추가 발견한 내용}
- **카테고리**: 누락방지 / 심층분석 / 하드코딩탐지 / 크로스체크 / 기타
- **검증**: ⬜ 미검증 / ⭐ 검증됨
```

## 누락 방지 패턴

### [예시] Step 03 — MyBatis XML 누락 탐지
- **서비스**: (예시)
- **상황**: DB 관련 항목이 ❌(해당 없음)으로 판정되었으나, 실제로는 MyBatis XML Mapper가 존재
- **효과적 프롬프트**:
  ```
  src/ 하위에서 *.xml 파일 중 <mapper 또는 <select 태그가 포함된 파일을 모두 찾아줘.
  sqlmap/, mapper/, mybatis/ 폴더도 확인해줘.
  ```
- **발견 결과**: `src/main/resources/sqlmap/` 하위에 Mapper XML 42개 발견
- **카테고리**: 누락방지
- **검증**: ⭐ 검증됨

### [예시] Step 05 — 개인정보 필드 간접 참조 탐지
- **서비스**: (예시)
- **상황**: 개인정보 처리 현황에서 일부 필드가 누락됨 (변수명이 약어로 되어 있어 탐지 실패)
- **효과적 프롬프트**:
  ```
  개인정보 관련 필드를 다시 확인해줘.
  phone, tel, mobile, hp, 전화, email, mail, name, nm, 이름,
  addr, address, 주소, ssn, jumin, 주민, ci, di, birth, 생년
  이런 약어/한글 변형까지 모두 grepSearch 해줘.
  ```
- **발견 결과**: `userNm`, `hpNo`, `birthDt` 등 약어 필드 15개 추가 발견
- **카테고리**: 누락방지
- **검증**: ⭐ 검증됨

## 하드코딩 탐지 패턴

### [예시] Step 01 — 하드코딩된 URL/크리덴셜 전수 스캔
- **서비스**: (예시)
- **상황**: 설정 프로파일 분석에서 하드코딩 항목이 일부만 식별됨
- **효과적 프롬프트**:
  ```
  소스코드 전체에서 아래 패턴을 grepSearch로 전수 스캔해줘:
  1. http:// 또는 https:// 가 포함된 문자열 리터럴 (설정파일 제외, .java/.kt/.js/.ts만)
  2. password, passwd, secret, api_key, apikey, token 이 포함된 대입문
  3. jdbc: 로 시작하는 문자열
  4. 숫자로만 된 포트번호 대입 (port = 8080 등)
  각각 파일 경로와 라인 번호를 정리해줘.
  ```
- **발견 결과**: 테스트 코드 외 프로덕션 코드에서 하드코딩 URL 8건, 크리덴셜 2건 추가 발견
- **카테고리**: 하드코딩탐지
- **검증**: ⭐ 검증됨

## 심층 분석 패턴

### [예시] Step 04 — 숨겨진 상태 전이 발견
- **서비스**: (예시)
- **상황**: enum 기반 상태만 추출되고, DB status 컬럼 기반 상태 전이가 누락됨
- **효과적 프롬프트**:
  ```
  DB 테이블에서 status, state, flag, type 컬럼을 가진 Entity를 모두 찾고,
  해당 컬럼 값을 변경하는 UPDATE 쿼리나 setter 호출부를 추적해줘.
  enum이 아닌 String/int 타입 상태값도 포함해줘.
  ```
- **발견 결과**: `order_status` 컬럼이 String 타입으로 7개 상태값을 가지며, 3개 서비스에서 직접 UPDATE
- **카테고리**: 심층분석
- **검증**: ⭐ 검증됨
