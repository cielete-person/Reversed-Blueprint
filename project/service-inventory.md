# 서비스 인벤토리

> Phase 0에서 작성. 대상 서비스 목록, 폴더명, Repository 정보, 분석 진행 현황을 관리한다.
> 플랫폼 관점 상세 정보(앱/단말/서버 관계)는 [platform-service-inventory.md](platform-service-inventory.md) 참조

---

## 서비스 ID 채번 규칙

- 형식: `SVC-{3자리 순번}`
- 순번은 `platform-service-inventory.md`의 순번과 동일
- 플랫폼 그룹: 미디어 / 인증 / 메시징 / 결제 / 광고마케팅 / IoT / 위치 / VoIP

## 폴더명 규칙

- 영문 소문자 + 하이픈(`-`) 사용
- 형식: `{플랫폼약어}-{서비스영문명}`
- 플랫폼 약어: `media`, `auth`, `msg`, `pay`, `adm`, `iot`, `loc`, `voip`

## 소스코드 clone 명령

```bash
# 서비스별 소스코드를 services/{폴더명}/src/ 에 clone
git clone {Repo URL} services/{폴더명}/src
```
