# Security Layer 설계/구현 체크리스트

> 출처: 보안기술팀 송준기 CTO
> 서비스 개발 시 설계/구현 준수내용을 확인하여 개발해야 하는 보안 체크리스트.
> 설계도 역추적 시, 각 서비스의 소스코드가 이 체크리스트를 준수하는지 대비 분석한다.
> 파묘 프로젝트의 보안 추출 항목은 [extraction-checklist.md](extraction-checklist.md) 1-7 참조

## 1. 인증/인가 및 권한 관리 (1~35)

| # | 요구사항/기능 | 설계/구현 준수내용 | 확인 |
|---|---|---|---|
| 1 | 인증대상 및 방식 설계 | 모든 사용자 및 컴포넌트에 대한 인증 대상 식별, 서버측 안전한 인증 방식 설계 | ⬜ |
| 2 | 서버 기반 인증 | 중요 기능은 인증 후 사용, 검증은 서버에서 중앙 집중식 처리 | ⬜ |
| 3 | 다중 요소 인증(MFA) | 핵심 기능 및 개인정보처리시스템에 OTP/생체 등 MFA 적용 | ⬜ |
| 4 | 인증 실패 로직 | 인증 실패 및 예외 로직 고려 설계 | ⬜ |
| 5 | 인증 수행 제한 기준 설정 | 반복 인증 시도 차단, Brute-force/사전 공격 방지 | ⬜ |
| 6 | 시도 횟수 제한 및 잠금 정책 | 계정 잠금 등 반복 인증 시도 차단 정책 | ⬜ |
| 7 | 공격 지연/차단 | 실패 시 CAPTCHA, 시간 지연 등 정책 | ⬜ |
| 8 | 실패 로깅 | 실패한 인증 시도 정보 상세 기록 | ⬜ |
| 9 | 중요 자원 접근통제 | 중요 자원 정의, 최소 권한 원칙과 RBAC 등 접근 통제 | ⬜ |
| 10 | 접근 권한 최소화 | ACL/RBAC 등 적용, 최소 권한 설정 | ⬜ |
| 11 | 자원 접근권한 확인 | 요청 자원이 해당 사용자에게 허용된 자원인지 서버 측 확인 | ⬜ |
| 12 | 관리자 영역 통제 강화 | 관리자 페이지 일반 사용자 노출 방지 | ⬜ |
| 13 | 미사용 계정 권한 관리 | 90일 이상 미사용 계정 비활성화/삭제 | ⬜ |
| 14 | 재인증 의무화 | 민감 상태 변경 전 본인확인/비밀번호 재입력 등 재인증 | ⬜ |
| 15 | 자원 접근 권한 검증 | 자원 ID가 현재 사용자 기준 조회·수정 가능한지 서버 확인 | ⬜ |
| 16 | 검증된 프레임워크 사용 | Spring Security 등 검증된 라이브러리로 접근 통제 | ⬜ |
| 17 | 서버 기반 권한 검증 | 세션·토큰·DB 등 서버 관리 정보 기준 권한 판단 | ⬜ |
| 18 | 중앙 집중식 인가 구조 | Filter/Interceptor/AOP 등으로 인가 로직 한 지점 처리 | ⬜ |
| 19 | 인가 우회 방지 | Path Variable/Query Parameter/URL 변조 등 우회 방지 | ⬜ |
| 20 | 내부 호출 경로 검증 | 모든 경로(컨트롤러, 서비스, 내부 API)에서 동일 인가 검증 | ⬜ |
| 21 | 관리 기능 접근 경로 통제 | 서버 권한 검증 + 허용 IP 대역 제한 | ⬜ |
| 22 | 중요 자원 최소 권한 설정 | 설정/실행/로그 파일은 관리자/프로세스 계정만 접근 | ⬜ |
| 23 | 기본 권한 생성 시 제한 | umask 등으로 불필요한 권한 부여 방지 | ⬜ |
| 24 | 쓰기 필요 자원만 예외 허용 | 쓰기 필요 파일만 명시적 허용 | ⬜ |
| 25 | 배포 시 권한 검증 절차 | 배포 과정에서 파일 권한 변경 여부 확인 | ⬜ |
| 26 | 로그인 실패 횟수 제한 | 최대 3~5회 초과 시 추가 인증/차단 | ⬜ |
| 27 | 일시적 계정 잠금 | 초과 시 일정 시간 잠금 | ⬜ |
| 28 | 추가 인증 적용 | 반복 실패 시 CAPTCHA/OTP 등 | ⬜ |
| 29 | 실패 시도 기록 관리 | 실패 이력 서버 관리, 초기화/잠금해제 정책 | ⬜ |
| 30 | 비밀번호 복잡도 강제 | 2종류 10자리 또는 3종류 8자리 이상, 서버 검증 | ⬜ |
| 31 | 취약 비밀번호 패턴 제한 | ID/연속숫자/생일/전화번호 등 차단 | ⬜ |
| 32 | 비밀번호 재사용 제한 | 최근 사용 비밀번호 재사용 불가 | ⬜ |
| 33 | 임시 비밀번호 변경 강제 | 최초 로그인 후 변경 완료해야 이용 가능 | ⬜ |
| 34 | 비밀번호 변경 주기 관리 | 3개월 1회, 대고객은 6개월 1회 | ⬜ |
| 35 | 비밀번호 재설정 추가 인증 | 이메일/SMS/OTP 등 별도 인증수단 | ⬜ |
