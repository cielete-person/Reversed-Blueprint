# 기획 용어 → 코드 구현체 매핑 사전 (Business–Tech Glossary)

> 서비스 기획자가 사용하는 비즈니스 용어와 실제 코드 구현체(클래스, API, 모듈, DB 테이블) 간 매핑.
> CR(변경 요청) 영향도 분석 시 "기획 용어 → 코드 위치 → 담당팀" 역추적에 활용.
> 기존 [glossary.md](../../glossary.md)는 프로젝트 공통 용어 사전이며, 이 파일은 기획↔코드 매핑에 특화.

---

## 축적 규칙

1. Phase 1 분석(Step 01~08) 중 기획 용어와 코드 구현체 간 매핑이 발견되면 이 파일에 추가
2. 각 항목에 `발견 서비스`, `발견 Step`, `검증 상태`를 기재
3. 동일 기획 용어가 여러 서비스에서 다르게 구현된 경우 모두 기재 (서비스별 행 분리)
4. 기획자/PO가 검증한 항목은 `검증 상태`를 ✅로 변경

---

## 매핑 테이블

| 기획 용어 | 코드 구현체 (클래스/API/모듈) | 서비스 | 담당팀 | 발견 Step | 검증 상태 |
|---|---|---|---|---|---|
| *(Phase 1 분석 시 축적)* | | | | | |

### 예시 (Phase 1 분석 후 채워질 형태)

| 기획 용어 | 코드 구현체 (클래스/API/모듈) | 서비스 | 담당팀 | 발견 Step | 검증 상태 |
|---|---|---|---|---|---|
| VOD 구매 | `PurchaseService.buyVod()`, `/api/v1/purchase/vod` | IPTV VOD | 미디어서비스개발팀 | Step 04 | ⬜ 미검증 |
| 본인확인 | `IdentityVerifyController`, `/api/v1/auth/verify` | PASS | 인증/결제플랫폼개발팀 | Step 03 | ⬜ 미검증 |
| 스팸 차단 | `SpamFilterEngine.filter()`, `spam_rules` 테이블 | 스팸차단 | 메시징플랫폼개발팀 | Step 04 | ⬜ 미검증 |
