# 코드 소유권 맵 (Code Ownership Map)

> 코드 영역(모듈/패키지/API 그룹) → 담당 조직(팀/파트) 매핑.
> 사람이 아닌 "조직 코드"로 매핑하여 조직재편에도 안정적으로 유지한다.
> 현재 담당자/외주사 정보는 [org-contact-directory.md](org-contact-directory.md) 참조.

---

## 매핑 원칙

1. 코드 영역은 `services/{서비스명}/` 폴더 단위로 1차 매핑
2. 서비스 내부 모듈/패키지는 Phase 1 분석 후 2차 매핑 (하위 테이블에 추가)
3. 담당 조직은 "팀" 단위 (파트/셀 단위는 org-contact-directory.md에서 관리)
4. 외주 개발사가 실제 개발하는 영역은 `[외주: {회사명}]` 태그로 구분
5. 네트워크부문 운영팀 등 타 부문이 관리하는 서버는 `[운영: {부문}]` 태그로 구분

## 관리 주체 구분

| 태그 | 의미 | 설명 |
|---|---|---|
| `[Lab]` | Consumer서비스개발Lab. 직접 관리 | 소스코드 접근 및 변경 권한 보유 |
| `[외주: 미디어로그]` | 미디어로그 외주 개발 | Lab 소속 팀이 PM, 미디어로그가 실제 개발 |
| `[운영: 네트워크부문]` | 네트워크부문 운영팀 관리 | 소스코드 접근 제한 가능성 있음 |
