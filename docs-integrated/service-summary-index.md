## 기술 부채 등급 기준

| 등급 | 기준 | 색상 |
|---|---|---|
| 🔴 Critical | Dead Code > 20% 또는 보안 Gap > 10 또는 테스트 커버리지 < 20% | 빨강 |
| 🟡 Warning | Dead Code 10~20% 또는 보안 Gap 5~10 또는 테스트 커버리지 20~50% | 노랑 |
| 🟢 Good | Dead Code < 10% 및 보안 Gap < 5 및 테스트 커버리지 > 50% | 초록 |

## Phase 3 횡단 분석 시 활용 방법

1. 이 인덱스를 먼저 로드하여 30개 서비스의 전체 현황을 파악
2. 기술 부채 등급 🔴 서비스를 우선 분석 대상으로 선정
3. 서비스 간 의존성 요약에서 허브 서비스(많은 서비스가 의존하는 서비스) 식별
4. 필요한 서비스의 상세 산출물만 선택적으로 로드하여 분석
